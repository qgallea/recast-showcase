# Changelog — Round 1 (Teacher Training)

Revision implemented against `synthesis.md` (Major revision: 1 blocking, 4
essential, 8 suggestions). Because the RECAST notebooks are generated by
`phase2/src/build_stage_notebooks.py` and the estimators live in
`recast/lib/recast_dml.py`, the fixes were made in the **generator and the
shared library** (so they cannot be lost on regeneration and they propagate to
future papers), then the stage-4→6 cascade was re-run. NT is unaffected
(`fixed_effects == []` ⇒ the blockZ logic is a no-op there; its frozen results
and passed regression gate are untouched).

## Blocking issue addressed

- **B1 [C6/C19] blockZ randomization strata omitted from every covariate set.**
  **Status: Resolved (code fix + stage-4→6 re-run).**
  - **Fix:** added `add_fixed_effects(df, ident, drop_first)` to the builder
    HEADER; it one-hot-encodes `paper_spec.identification.fixed_effects` and
    appends the dummies to the inherited conditioning set. Now used in:
    - Stage 3 replication OLS (drop-first to avoid intercept collinearity) —
      blockZ enters the ITT regression.
    - Stage 4 IRM nuisances **and** the CDDF generic-ML proxy covariate set
      (all 6 levels) — so B(Z)/S(Z) condition on the strata.
  - **Re-run result (reported honestly, NOT tuned):** β₂ (NeuralNet, best)
    moved only 0.178 → **0.1996** (CI90 [−0.310, 0.695], p = 0.560); the gap to
    B&N's 0.651 persists ("divergent"). The ITT moved 0.0066 → **0.0018**
    (toward the benchmark 0.002, both null). `dml_results.n_features_raw` is now
    30 (24 controls + 6 blockZ dummies). **Interpretation:** restoring the
    pre-registered strata did not close the β₂ gap, so candidate explanation
    (c) is *eliminated* — the implementation-sensitivity finding is sharpened,
    not the benchmark chased. Blinding preserved: the extension notebook never
    read `benchmark_results.json`; the orchestrator recomputed the gap table
    after `dml_results.json` was frozen.

## Essential issues addressed

- **E1 [C3] n_rep = 20 disclosure.** Status: Resolved (prose, in stage-6
  regeneration). The Boosting MC-dispersion qualifier is added; the disclosed
  pre-registered justification stands; the benchmarked CDDF quantity keeps the
  full 100-split protocol. (Per-cell budget did not permit lifting IRM n_rep to
  100 within the timeout; the disclosure is the accepted resolution.)
- **E2 [C9/R2b] propensity diagnostics.** Status: Resolved (lib + report).
  `recast_dml.doubleml_model` now records, for IRM, `propensity_diag`
  (min/max, trimmed share) and `nuisance_loss_metric` (ml_m = logloss, not
  RMSE). Re-run values: Forest m̂∈[0.022, 0.974], Boosting m̂∈[0.010, 0.984],
  trimmed share 0.0. The stage-6 report relabels the propensity column as
  log-loss and notes the learners underperform the known constant p = 0.513
  (expected under school-level randomization).
- **E3 [C1/C14] `interactions_used: true` contradicted the executed code.**
  Status: Resolved (builder). `interactions_used` is now conditioned on a
  penalized learner actually running; for this IRM/tree run it is **False**
  with `interactions_note` explaining tree learners used the raw controls and
  no interaction basis was built.
- **E4 [C16] variational CI coverage mislabel.** Status: Resolved (builder +
  lib usage). The CDDF call now passes per-split `alpha = 0.05`, so the
  medianed bounds are genuine **90%-adjusted** CIs (coverage ≥ 1 − 2α = 90%),
  matching B&N's convention; `hte_results.cddf.ci_convention` documents it.
  CIs widened slightly (e.g. β₂ [−0.287, 0.639] → [−0.310, 0.695]); the
  homogeneity non-rejection is unchanged.
