# General Referee Report

**Round:** 1
**Overall verdict:** Minor revision

### Contribution assessment

This RECAST adds real value on three fronts: (i) a blinded, independently implemented head-to-head of the CDDF generic-ML protocol against Baiardi & Naghi (2024) on the same prepared sample, with the gap table assembled only after `dml_results.json` was frozen (SHA prefix `92c007fe502475a7` matches `gap_table.json`); (ii) a design-faithful school-level-splitting variant of generic-ML inference for a cluster-randomized trial, which is itself a methodological data point; and (iii) an IRM ATE extension confirming the published null under flexible nuisances. The headline product — the same protocol on the same data yielding β₂ = 0.178 (p = 0.581) where the benchmark reports 0.651 (p = 0.0003), with the rank-dependent outputs (GATES shape, 9/10 CLAN directions) reproducing and the scale-dependent ones not — is a genuinely informative finding about implementation sensitivity, framed without benchmark-chasing. **Pleased-to-have-produced test: yes, flaws and all.**

### Essential issues (must be fixed before the result can be trusted)

1. **BlockZ randomization-strata fixed effects: spec-declared, apparently excluded, only partially disclosed.** The spec declares `fixed_effects: ["blockZ"]`, and its `_blocks_note` states blockZ is "a valid stratification control (entered as fixed effects / one-hot in nuisances)." The evidence indicates it was not entered anywhere: `dml_results.json` reports `n_features_raw: 24` (the 24 controls only, before interactions), so the IRM nuisances exclude it; Section 2.2 and Table 1 describe the replication OLS as "the treatment dummy and the 24 inherited baseline controls," silent on strata FEs; and Section 5.5(c) discloses the exclusion only for the generic-ML proxies. This sits in tension with the report's own inheritance claim (estimand statement: conditioning set inherited "variable-for-variable") and with Table 1's title "Replication of the original ITT specification" — in a stochastic regime with no deterministic target, specification fidelity is the only replication check available, so the reader must be able to verify it. To be clear about direction: with county-stratified randomization the strata are balanced by design, so omitting them cannot bias the ATE point estimate (precision-only effect) — the null ATE conclusion is not at risk. But the report cannot simultaneously claim variable-for-variable inheritance and silently drop a spec-declared element of the conditioning set.
   *Action:* State explicitly, in Section 2.2 and Section 6, whether blockZ dummies entered (i) the replication OLS and (ii) the IRM nuisances. If excluded (as the JSONs indicate), either add them and re-run the cheap pieces (the OLS rerun is near-free), or keep the exclusion and justify it in the text with the precision-only argument, amending the "variable-for-variable" language accordingly. No re-run of the 100-split generic-ML protocol is required for this item (see Suggestion 2).

### Suggestions (non-binding)

1. **Fix the estimand-statement provenance wording.** `estimand_statement.md` (embedded verbatim in Section 3) says the 24 controls are inherited "variable-for-variable from the original code," but the declared regime is **port** with "no original code on disk" (Section 2.1, `replication_results.json`). The controls actually come from B&N's variable-description PDF, and per the spec's `_control_notes` some were added by B&N beyond the original Loyalka set. Reword to "variable-for-variable from the documented B&N specification" to remove the internal contradiction.

2. **One informative sensitivity check, if compute allows.** Candidate explanation (c) — blockZ excluded from the proxy covariates — is directly testable: re-run the generic-ML protocol for the selected NeuralNet learner only, with blockZ one-hot added to the proxy covariate set, clearly labeled as a disclosed post-hoc sensitivity (not tuning: the variant is motivated by the spec's own `_blocks_note`, not by the benchmark value). This is the single most informative check available; restricting to one learner keeps cost bounded. If β₂ remains insignificant, candidate (c) is eliminated and the implementation-sensitivity finding sharpens; if it becomes significant, the divergence is explained.

3. **Add the second verdict layer to the gap table.** `gap_table.json` rows carry a single verdict label; the spec's benchmark-mode format calls for two layers (materiality + MC-implementation flag). Here the second layer would mostly read "benchmark implementation unverifiable (no B&N code on disk)" — which is exactly the point worth stamping on the β₂ row. Also fix the truncated benchmark strings in the two IRM rows of `gap_table.json` ("…our IRM ATE ro"); the paper renders them correctly but the artifact is malformed.

4. **Confirm the β₁ benchmark cell.** The gap table's BLP-β₁ benchmark (0.002, CI90 [−0.068, 0.072], p = 1.0) is digit-for-digit identical to the OLS benchmark row. If B&N's reported BLP β₁ genuinely coincides with their OLS estimate at this precision, say so in a table note; if the orchestrator reused the OLS row as a proxy for β₁, label it as such.

5. **Note the median-aggregation arithmetic in Table 5.** The CLAN "Diff." column is the median-over-splits difference, which need not equal the difference of the median group means (e.g., `female`: −0.080 reported vs −0.084 implied by 0.446 − 0.530; both correct per `hte_results.json`). A one-line table note would pre-empt the appearance of an arithmetic error.

### Comments

This is a disciplined report. The replication situation is stated with unusual honesty: the stochastic regime, the PARTIAL tier with its named explanation (identical in `replication_check.json`, `replication_results.json`, and Table 1), the port declaration, and the inherited cluster SE convention are all consistent across text, tables, and JSONs. Number traceability is excellent — every BLP, GATES, CLAN, IRM, and diagnostic figure in the paper matched its JSON source on my checks, the verdict counts (11 consistent / 1 divergent / 1 shape-consistent / 1 direction-flipped / 2 no-benchmark) reconcile with the 16 table rows, and sample sizes (10,006; 9,676 complete cases; 201 clusters; treated share 0.513 = the recorded propensity 0.51299) are consistent everywhere. The β₂ divergence is handled exactly as one would hope: stated plainly and prominently, candidate explanations labeled as unadjudicated hypotheses, the attenuation mechanism's direction argued explicitly (all candidates make the RECAST test more conservative, not the benchmark invalid), and the power-honesty paragraph declines to claim either detection or refutation — noting the 90% CI [−0.287, 0.639] contains most of the benchmark's [0.312, 0.99]. Suspect-control discipline holds: `suspect_controls` is empty with a documented screen (including the post-treatment check on `t_training_hours_terc`), the forbidden list correctly quarantines all endline/midline/attrition variables and the treatment-leaking `blockZZ`, and none appears in any estimation. Moderators are the 10 pre-declared baseline variables; causal language is proportionate to the RCT design; the n_rep = 20 deviation is disclosed with its pre-registered justification (the mechanics of that deviation are the technical referee's lane, and the disclosure is exemplary).

The one essential item is narrow and cheap: the blockZ strata are part of the declared conditioning set, and the report's inheritance claim must be reconciled with their apparent exclusion from the replication and the IRM nuisances — disclosure plus either inclusion or a stated precision-only justification suffices; the null-ATE conclusion is not in jeopardy. With that fixed and the wording suggestion (Suggestion 1) addressed, the report's claims are proportionate to its evidence and the divergence framing can be trusted as-is.
